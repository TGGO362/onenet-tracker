OneNet + Vercel + 高德地图 12 小时轨迹系统（修订版）
一、整体架构
IoT 设备 → OneNet 平台 → Vercel Serverless 函数 → Vercel KV（仅存近 12 小时经纬度）→ Vercel 静态网页（高德地图渲染 + 时段选择）
核心目标：存储近 12 小时时序位置，网页端可选 1~12 小时时段，实时渲染点位与轨迹，逻辑极简、无额外防护与缓存。
二、技术选型
地图：现有高德地图 JS API
托管：Vercel（静态页面 + Serverless 函数）
存储：Vercel KV（设置数据 TTL=12 小时，自动清理过期数据）
数据来源：OneNet 开放 API（主动轮询拉取数据）
外部依赖：无 Token 鉴权、无前端数据缓存
三、各模块设计调整与说明
3.1 OneNet 平台配置
提取设备 ID、API 密钥、经纬度对应数据流字段，开启数据读取权限。
设备上报间隔建议 30s~2min，控制点位总量（12 小时内点位数量远低于 40 个，数据体量极小）。
无需配置消息推送、IP 白名单，仅使用开放 API 轮询。
3.2 Vercel Serverless 函数（两个接口）
接口 1：数据采集接口
定时请求 OneNet API，拉取最新经纬度 + 时间戳。
数据清洗：过滤无效坐标、重复静止点位。
写入 Vercel KV，单条数据设置 12 小时 TTL，过期自动删除，永久只保留近 12 小时数据。
补充：Vercel 无内置定时，使用第三方轻量定时工具轮询调用该接口即可。
接口 2：轨迹查询接口
接收前端传入参数：选择的时长（1/3/6/9/12 小时）。
根据当前时间计算时间阈值，从 KV 中筛选对应时段内的所有点位。
按时间升序整理数据，直接返回给前端。
取消 Token 鉴权：依靠 Vercel 随机后缀域名做天然隔离，个人场景无安全风险。
环境变量配置
OneNet 密钥、高德地图密钥统一存入 Vercel 环境变量，不暴露在前端代码。
3.3 Vercel KV 存储结构
Key：设备 ID + 时间戳（唯一标识单条位置数据）
Value：JSON 格式，包含经度、纬度、采集时间戳
过期规则：固定 TTL=12h，自动淘汰历史数据，无需人工维护
3.4 前端网页（核心调整：移除缓存）
基础能力：初始化高德地图，展示最新设备点位与坐标信息。
交互控件：提供时段选择器（1h/3h/6h/9h/12h）。
数据请求逻辑：每次切换时段，立即发起全新后端请求，不做本地数据缓存，保证每次查看都是最新轨迹。
地图渲染：
按返回点位顺序绘制标记点与轨迹折线；
地图自动适配视野，完整展示所选时段轨迹；
异常处理：无数据 / 设备离线时，给出文字提示，展示最后一条有效位置。
四、完整数据流转
物联网设备定时上报经纬度至 OneNet；
定时任务触发 Vercel 采集接口，拉取数据、清洗后存入 KV（12 小时自动过期）；
用户访问网页，加载高德地图；
用户选择轨迹时段，前端直接请求查询接口；
后端筛选对应时段数据并返回；
前端解析数据，在地图上绘制点位与轨迹。
五、关键优化与注意事项
性能说明：单条查询数据量＜40 个点位，单次请求开销极低，频繁直连后端无性能压力，逻辑大幅简化。
安全策略：沿用 Vercel 随机域名后缀，不额外增加鉴权逻辑，匹配个人使用场景。
数据实时性：彻底移除前端缓存，保证每次操作都获取服务端最新数据，符合偶尔查看、需要最新轨迹的使用习惯。
点位优化：后端依旧过滤重复静止点位，避免轨迹线条冗余杂乱。
超时适配：控制 Serverless 函数执行时长，满足 Vercel 运行限制。
六、最终落地组合
OneNet API 轮询 + Vercel 静态页 + Vercel Serverless 函数 + Vercel KV（12h 自动过期）+ 高德地图 API
特点：架构轻量化、代码逻辑简单、免运维、免费资源全覆盖，完全适配个人低频使用场景。
